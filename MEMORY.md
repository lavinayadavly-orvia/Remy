# MEMORY.md

## Platform Identity
- Platform name: MyDesk
- MyDesk V1 is a private, local-first executive execution assistant for one senior healthcare/RWE/business strategy user.
- MyDesk is not a chatbot, backend utility, crude prototype, project management dashboard, or bare-minimum MVP.
- MyDesk should let the user delegate a serious business task, walk away, and return to a polished downloadable output that looks and sounds like something the user would create.
- MyDesk learns from the user's decks, proposals, templates, offering documents, references, scope language, slide structures, color palette, strategy patterns, and executive phrasing.
- MyDesk should ask only when material information is missing, avoid exposing internal workflow steps, and keep files local-first and confidential.

## Operating Rules
- No filler openings.
- Significant tasks require 2-3 approaches before execution.
- Ask before assuming when material information is missing.
- Never present uncertain facts as confirmed.
- Only change what the user specifically asks to change.
- Confirm before major content changes.
- Confirm before irreversible or external side-effect actions.
- Keep responses direct, practical, senior, and concise.

## Fixed Project Facts
- Platform name: MyDesk.
- Product framing: MyDesk V1 - a private, world-class executive execution assistant for one senior healthcare/RWE/business strategy user, built cleanly so it can scale later.
- V1 pillars: Private Library, Learning Engine, Command Interface, Output Generator, Preview + Download, History, Error/quality checks.
- V1 exclusions: no auth, payment, team sharing, cloud database, email delivery, exposed workflow, fake placeholder content, or generic AI output.

## Decisions
- 2026-05-12: The platform was renamed from StudioRWE Workbench to MyDesk because the user specified MyDesk as the fixed platform name.
- 2026-05-12: Placeholder local generation was rejected because it created misleading scaffold content when OpenAI was not configured.
- 2026-05-12: The main workbench UI was simplified around Command in -> Plan -> Draft -> Critique -> Final -> Export -> Next action, using a clean header, left command/materials panel, middle stage output panel, and right review/action panel.
- 2026-05-14: The workbench UI was refreshed as a guided execution surface: left command/materials, center output stage workspace, and right review/refine/export actions, with clearer empty states and stage progression.
- 2026-05-15: MyDesk added a template-aware generation input. Word/PowerPoint templates are uploaded separately from reference materials and used as structure/style guidance during planning, without changing the export pipeline yet.
- 2026-05-15: MVP scope was reset to Prompt -> Generate -> Preview -> Download. Do not implement email delivery in MVP, do not build complex workflow panels, do not require repeated uploads, hard-error when OpenAI is not configured, save run history locally, keep generated files under `outputs`, log errors in ERRORS.md, and log key decisions here.
- 2026-05-15: Lower-level AI fallback generation paths were removed from the active pipeline. If OpenAI is unavailable or model JSON is invalid, MyDesk must fail clearly instead of returning deterministic placeholder content.
- 2026-05-18: Project workspace UI was reset to one clean command screen: command, output type, generate final output, then preview and the matching download button. Internal workflow panels and approach selection must not interrupt normal MVP generation.
- 2026-05-18: PowerPoint output must pass a deck-specific quality gate before save/export. Decks need action headlines, one clear message per slide, evidence caveats for exact data, no placeholders, no "Generated by MyDesk" in slides, limited bullets, and structured boardroom layouts.
- 2026-05-18: Product direction was reframed from bare-minimum MVP to MyDesk V1: a premium private executive execution assistant for one senior healthcare/RWE/business strategy user. V1 must feel polished and reliable while staying local-first, single-user, modular, and free of SaaS features for now.
